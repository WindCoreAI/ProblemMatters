# Industry Taxonomy Hierarchy

**Version**: 1.0.0
**Status**: Reference Document
**Last Updated**: 2026-01-20

---

## Overview

This document provides a comprehensive hierarchical taxonomy of industries, domains, fields, and subfields for the Problem Matters platform. The taxonomy is designed to:

1. Enable precise problem classification
2. Support intuitive navigation and discovery
3. Align with standard industry classifications (NAICS, SIC, GICS)
4. Cover both traditional and emerging sectors
5. Balance breadth with practical depth

---

## Taxonomy Structure

```
Industry (Level 1)
└── Domain (Level 2)
    └── Field (Level 3)
        └── Subfield (Level 4)
```

**Depth Guidelines**:
- Every problem must have Industry and Domain
- Field is recommended for specificity
- Subfield is optional for highly specialized problems

## Usage Guidelines

### Classification Rules

1. **Primary Classification**: Each problem gets ONE primary industry, domain, and optionally field
2. **Specificity**: Use the most specific applicable classification
3. **Cross-Industry**: Problems spanning multiple industries should use the primary affected industry
4. **Emerging Areas**: New fields can be added following the governance process
5. **Deprecation**: Old classifications are marked deprecated, not deleted

### Adding New Classifications

1. Propose new classification with rationale
2. Review against existing taxonomy for overlap
3. Provide NAICS/GICS mapping if applicable
4. Document parent-child relationships
5. Update search indices and filters

### Search & Discovery

- All classification levels are searchable
- Problems inherit search terms from parent classifications
- Synonyms and aliases are maintained in a separate mapping file
- Industry codes (NAICS, GICS) are indexed for professional users
